═══════════════════════════════════════════════════════════════
  СПОСОБ 1: Автоматическая отправка (РЕКОМЕНДУЕТСЯ)
═══════════════════════════════════════════════════════════════

Запустите файл: push_to_remote.bat

Скрипт автоматически:
- Проверит наличие удаленного репозитория
- Если его нет - попросит ввести URL
- Отправит коммиты в ветку main

═══════════════════════════════════════════════════════════════
  СПОСОБ 2: Ручная отправка через командную строку
═══════════════════════════════════════════════════════════════

1. Создайте репозиторий на GitHub/GitLab/Bitbucket

2. Добавьте удаленный репозиторий:
   git remote add origin https://github.com/username/repository.git
   
   ИЛИ для SSH:
   git remote add origin git@github.com:username/repository.git

3. Отправьте коммиты:
   git push -u origin main

═══════════════════════════════════════════════════════════════
  ВАЖНО ДЛЯ GITHUB
═══════════════════════════════════════════════════════════════

GitHub больше не принимает пароли при push.
Используйте Personal Access Token:

1. GitHub → Settings → Developer settings → Personal access tokens
2. Создайте новый token с правами "repo"
3. Используйте token вместо пароля при аутентификации

Или настройте SSH ключи для более удобной работы.
